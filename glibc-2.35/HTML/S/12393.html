  46 // API

  48 // long double cbrtl(long double)
  49 //
  50 // Overview of operation
  51 //==============================================================
  52 // Background
  53 //
  54 // Implementation
  55 //
  56 // The result is computed as
  57 // cbrt(x)= cbrt(1 - (1 - x*y)) * (1/cbrt(y))
  58 // where y = frcpa(x) = (-1)^sgn_y * 2^(3*k+j) * m_y,
  59 //       m_y in [1,2),  j in {0,1,2}
  60 //
  61 //  cbrt(1 - (1 - x*y)) is approximated by a degree-6 polynomial
  62 //  in r= 1 - x*y :
  63 // P = 1 - c_1 r - c_2 * r^2 - c_3 * r^3 - c_4 * r^4 - c_5 * r^5 - c_6 * r^6
  64 //
  65 //
  66 // The values (1/cbrt(y)) are stored as two tables of constants T_hi
  67 // (double-extended precision) and D (single precision) as follows:
  68 // T_hi (1 + D)= 1/cbrt(y) to about 80 bits of accuracy
  69 //
  70 // The tables are only stored for three exponent values (i.e.
  71 // only for 2^j * m_y, where j in {0,1,2} and m_y covers the 256
  72 // possible mantissas for an frcpa result); the index is formed
  73 // by the 8 leading mantissa bits of x, which is the same index used
  74 // by the hardware to get frcpa(x).
  75 //
  76 // The table values are multiplied by 2^k where e is the exponent of
  77 // the input number.  This multiplication is carried out in parallel with
  78 // the polynomial evaluation:
  79 // T= 2^(k) * T_hi
  80 //
