  47 /* Linux uses a negative return value to indicate syscall errors,
  48    unlike most Unices, which use the condition codes' carry flag.
  49 
  50    Since version 2.1 the return value of a system call might be
  51    negative even if the call succeeded.  E.g., the `lseek' system call
  52    might return a large offset.  Therefore we must not anymore test
  53    for < 0, but test for a real error by making sure the value in %eax
  54    is a real error number.  Linus said he will make sure the no syscall
  55    returns a value in -1 .. -4095 as a valid result so we can savely
  56    test with -4095.  */

 116 /* Linux takes system call arguments in registers:
 117 
 118     syscall number  %eax         call-clobbered
 119     arg 1       %ebx         call-saved
 120     arg 2       %ecx         call-clobbered
 121     arg 3       %edx         call-clobbered
 122     arg 4       %esi         call-saved
 123     arg 5       %edi         call-saved
 124     arg 6       %ebp         call-saved
 125 
 126    The stack layout upon entering the function is:
 127 
 128     24(%esp)    Arg# 6
 129     20(%esp)    Arg# 5
 130     16(%esp)    Arg# 4
 131     12(%esp)    Arg# 3
 132      8(%esp)    Arg# 2
 133      4(%esp)    Arg# 1
 134       (%esp)    Return address
 135 
 136    (Of course a function with say 3 arguments does not have entries for
 137    arguments 4, 5, and 6.)
 138 
 139    The following code tries hard to be optimal.  A general assumption
 140    (which is true according to the data books I have) is that
 141 
 142     2 * xchg    is more expensive than  pushl + movl + popl
 143 
 144    Beside this a neat trick is used.  The calling conventions for Linux
 145    tell that among the registers used for parameters %ecx and %edx need
 146    not be saved.  Beside this we may clobber this registers even when
 147    they are not used for parameter passing.
 148 
 149    As a result one can see below that we save the content of the %ebx
 150    register in the %edx register when we have less than 3 arguments
 151    (2 * movl is less expensive than pushl + popl).
 152 
 153    Second unlike for the other registers we don't save the content of
 154    %ecx and %edx when we have more than 1 and 2 registers resp.
 155 
 156    The code below might look a bit long but we have to take care for
 157    the pipelined processors (i586).  Here the `pushl' and `popl'
 158    instructions are marked as NP (not pairable) but the exception is
 159    two consecutive of these instruction.  This gives no penalty on
 160    other processors though.  */
 161

 164     PUSHARGS_##args                               \
 165     DOARGS_##args                                 \

 167     ENTER_KERNEL                                  \
 168     POPARGS_##args
